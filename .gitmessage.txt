# Введите сообщение коммита
# Используйте один из следующих типов:
# feat: новая функциональность
# fix: исправление багов
# refactor: изменение кода без добавления новой функциональности
# docs: изменения в документации
# style: исправления стиля кода
# test: добавление/изменение тестов
#
# Пример:
# feat: добавлена новая возможность авторизации через Google
# fix: исправлена ошибка с загрузкой аватарки
#
# Заголовок (первое предложение) должно быть кратким и ясно объяснять суть изменений.
# Описание (если есть) может быть детализированным.
